# Plasma Instrument - Руководство по развёртыванию

## Структура проекта

Проект разделен на несколько основных компонентов:

```
plasma-instrument/
├── client/              # Фронтэнд (статический сайт + CMS интеграция)
├── server/              # Бэкэнд (CMS, админка, API)
├── build/               # Утилиты сборки
├── .deprecated/         # Архив старых статических статей (deprecated)
└── .git/               # Репозиторий
```

## CMS - Content Management System

С апреля 2026 года проект использует CMS для управления контентом:

- **Данные статей**: `server/storage/data/articles.json`
- **API бэкэнда**: работает на http://localhost:3000
- **Админка**: http://localhost:3000 (для редактирования контента)
- **Загрузки**: `server/storage/uploads/` (изображения и файлы)

### Запуск CMS

```bash
# В папке server
npm install
npm start
# Откройте http://localhost:3000 в браузере
```

## Запуск фронтэнда локально

Фронтэнд находится в папке `client/` и получает контент из CMS.

### Простой способ (локальный сервер)

```bash
cd client
npx http-server
# или
python -m http.server 8000
```

Откройте http://localhost:8000

## Генерация поискового индекса

После добавления новых статей через админку:

```bash
# Из корня проекта
node build/generate-search-index.mjs
```

Это обновит `client/search-index.json` с новыми статьями из CMS.

## Для разработчиков

### Добавить новую статью

1. Откройте админку: http://localhost:3000
2. Нажмите "Добавить новую статью"
3. Выберите раздел (blog, services, projects, sources)
4. Заполните название, контент, загрузите изображение
5. Сохраните

Статья сразу появится на фронтэнде!

### Добавить новую страницу (не статью)

Для постоянных страниц (не контент-ориентированные), которые не должны редактироваться в админке:

1. Создать папку в `client/pages/{имя_страницы}/`
2. Создать `index.html` с нужным контентом
3. Убедиться что используется правильный `data-depth`
4. Добавить ссылку в навигацию в `client/scripts/browser/components/header.html`

### Структура страницы

Используйте как шаблон: `client/en/index.html`

Обязательные элементы:
- `<html lang="ru" data-depth="2">` (depth = количество `../` до корня)
- `<title>` (для поиска и SEO)
- `<main class="section">` (контент)
- `<link rel="stylesheet" href="../../styles.css">`
- Скрипты подключения компонентов в конце body

## Развёртывание

### Фронтэнд
Разверните папку `client/` на любом статическом хостинге (Netlify, Vercel, GitHub Pages и т.д.)

Убедитесь что бэкэнд API доступен по адресу, указанному в `client/scripts/browser/cms-content.js` (переменная `API_BASE`)

### Бэкэнд
Разверните папку `server/` на хостинге с поддержкой Node.js

Убедитесь что переменная окружения указывает правильный PORT

## Миграция старых сайтов

Если вы переносите сайт с другой платформы на эту:

1. Экспортируйте старый контент
2. Подготовьте JSON в формате статей (см. `server/storage/data/articles.json`)
3. Используйте скрипт импорта: `node build/import-static-to-cms.mjs` (или создайте собственный)
4. Проверьте в админке
5. Опубликуйте

## Решение проблем

**"Cannot GET /api/..."**
- Убедитесь что бэкэнд запущен: `npm start` в папке `server/`

**Поиск не работает**
- Обновите индекс: `node build/generate-search-index.mjs`

**Изображения не загружаются**
- Проверьте что папка `server/storage/uploads/` создана и доступна
- Убедитесь что права доступа правильные
