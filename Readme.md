# Технологический стек
## Frontend
- HTML5, CSS3, Vanilla JavaScript (ES modules)
- Динамическая отрисовка контента через API (`/api`)
- DOMPurify для санитаризации HTML-контента
- Quill (в админке, подключение через CDN)

## Backend
- Node.js
- Express
- Multer (загрузка изображений)
- CORS
- dotenv

## Хранение данных и файлов
- JSON-хранилище статей: `server/storage/data/articles.json`
- Загруженные файлы: `server/storage/uploads/`

## Build и утилиты
- Генерация поискового индекса: `build/generate-search-index.mjs`
- Синхронизация индекса после CRUD-операций со статьями на сервере

# Структура проекта
## Где что находится
- `client/` — публичная часть сайта (страницы, стили, браузерные скрипты, поиск, компоненты header/footer).
- `client/admin/` — админка (логин и интерфейс управления статьями).
- `client/scripts/browser/` — основные фронтенд-модули (инициализация сайта, рендер CMS-контента, UI-компоненты).
- `server/` — API и раздача статики.
- `server/routes/` — маршруты API (статьи, разделы, загрузка, health-check).
- `server/middleware/` — авторизация админа и CSRF-проверки.
- `server/storage/` — JSON-данные и загруженные изображения.
- `build/` — скрипты сборочных/служебных задач (генерация поискового индекса).

## Как работает поток контента
1. Админка сохраняет статью через API (`/api/articles`).
2. Сервер пишет изменения в `server/storage/data/articles.json`.
3. После изменений сервер запускает генерацию `client/search-index.json`.
4. Публичный сайт читает статьи через API и отображает их на страницах разделов и статьи.

# Быстрый старт
## Локально
1. Запускаем сервак:
```bash
cd server
npm install
cp .env.example .env
# в .env задаем ADMIN_BASIC_AUTH_USERNAME/ADMIN_BASIC_AUTH_PASSWORD
npm start
```
2. Открываем сайт в браузере и вводим в адресную строку:
```text
http://localhost:3000/
```

## Админка
1. Открываем в браузере:
```text
http://localhost:3000/admin/
```
2. Если вход не выполнен, система перенаправит на страницу логина.
3. Вводим логин и пароль из файла `server/.env`.
4. После входа откроется админ-панель. В ней можно добавлять, редактировать и удалять статьи каждого раздела.

Если нужен отдельный статический сервер для клиента, можно дополнительно запустить `npx http-server -p 8080`, но для админки и API удобнее использовать `http://localhost:3000/`.

## Продакшен
1. Разворачиваем `server/` на Node.js-хостинге.
2. Разворачиваем `client/` на статическом хостинге.
3. Если сайт и API на разных доменах, указываем адрес API: в файле `client/scripts/browser/api-base-config.js` задаем значение `API_BASE_OVERRIDE`, например: `https://api.example.com/api`.
4. Если сайт и API на одном домене, оставляем `API_BASE_OVERRIDE` пустым — будет использоваться относительный путь `/api`.

## Полезно знать
- Данные контента хранятся в `server/storage/data/articles.json`.
- Загруженные файлы лежат в `server/storage/uploads/`.
- Если после правок контента нужен актуальный поиск, запускаем `node build/generate-search-index.mjs` из корня проекта (может пригодится только после прямых правок).
